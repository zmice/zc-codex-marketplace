# zc:ctx-health

这是 Codex 的 command-alias skill。

使用方式：

- 在 Codex 中直接调用 `$ctx-health`
- 它对应统一命令语义 `zc:ctx-health`
- 如果需要更深的方法细节，再继续调用相关专题 skill

这是 `command:start` 评估后进入的专项入口：当任务核心不是继续实现，而是先修复会话上下文质量时，转到 `ctx-health`。

调用 `context-engineering`，检查当前会话是否开始失焦、膨胀或混杂，并给出整理方案。

## 何时直接用这个入口

- 长会话后开始丢信息、跑题或重复劳动
- 准备切任务、切阶段，想先压缩上下文
- 需要判断是继续当前会话，还是新开会话/拆给子任务

## 何时回到 lifecycle

- 上下文整理完成，需要继续做需求或实现：回到 `spec`、`task-plan` 或 `build`
- 整理后发现真正问题是缺陷排查：转到 `debug`
- 如果要做更细的 token/组件占用分析：转给 `context-budget-audit`

## 执行重点

1. 检查上下文是否接近预算上限
2. 识别过时、无关、重复或互相冲突的信息
3. 输出结构化摘要：已完成、进行中、待办、关键决策、偏差点
4. 判断是否需要压缩总结、新会话或委派子任务
